TL;DR

Bulk feedback, minus the spreadsheet-shaped misery

Upload a Moodle submissions ZIP + your feedback template. Optionally add the grading CSV to pull email / StudentID / Grade. Then generate a ZIP Moodle can bulk-upload.

Privacy: everything runs in your browser. No uploads to any server.
Step 1 — Moodle submissions ZIP required
  • Use the original ZIP downloaded from Moodle using the Actions > Download Submissions in the assessment area.
  • The app reads folder names to extract Name + Moodle ID.
  • The table preview lets you sanity-check the cohort and submitted files.
The utiltiy currently doesn't suppor anonymous marking format. If the student folders don’t follow the expected Moodle pattern, IDs may not parse correctly.
Step 1b — Grading sheet CSV optional
  • Matches CSV Submission id ↔ Moodle ID from the ZIP.
  • Pulls Username as Email.
  • Splits at @ to get StudentID.
  • Pulls Grade.
  • CSV columns appear in the table only after a CSV is loaded.
Step 2 — Word template (.docx) required
{{Name}} {{MoodleID}} {{Filename}} {{Email}} {{StudentID}} {{Grade}}
  • Put tags anywhere in the document — the app replaces them when generating each file.
  • The utility will scan the template and reports which tags it detected.
  • Tags are case-sensitive: {{Name}} is different from {{name}}.
FAQ / gotchas
Why are the CSV columns hidden?

To keep the table clean. They appear only after a grading CSV is loaded.

Copy-to-Excel doesn’t work?

The modern clipboard API needs https:// or localhost. If you run the file directly, the fallback tries to copy anyway, but browsers vary.

My template shows “No {{tags}} detected.”

Either there are no tags, or Word has split them in a way that hides them from detection. Generation can still work — it’s just a warning sign.

Reset button
  • Use Reset to clear everything for a new module/assignment.
  • It clears loaded ZIP/CSV/DOCX state, hides the table preview, and disables Generate.
  • It does not touch any files on disk
Tip: if you’re doing multiple modules in a row, hit Reset first so you don’t accidentally generate feedback using an old template/CSV.
